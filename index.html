ECM Migration Tool Suite — DocOrbit Built at Tiera
Documentum Exporter · Alfresco Importer · FileNet · XECM

A pair of JavaFX desktop applications — an Exporter and an Importer — designed to enable organisations to migrate content between enterprise ECM platforms. The Exporter is built to pull documents, metadata, version history, and folder structures out of Documentum; the Importer is built to push them into Alfresco, FileNet, or XECM. The core Spring Batch pipeline is shared across all platforms — only the REST API integration layer is swapped per target system.

  • Multi-threaded Spring Batch export pipeline — authenticates against Documentum REST API, recursively traverses cabinet/folder trees, packages documents with full metadata and version trees into a structured XSD-validated XML manifest
  • Configurable chunked file streaming via Spring's ResponseExtractor — designed to split large files (500 MB+) into SHA-256-verified .part files to prevent OutOfMemoryError under concurrent batch threads
  • Two-step import pipeline: Step 1 reads a JAXB-unmarshalled transformation XML mapping source types to target equivalents; Step 2 streams records via StaxEventItemReader, creates Alfresco folder hierarchies dynamically, uploads with version history and XFDF annotation support
  • Visual Transformation UI — users configure source-to-target type and attribute mappings via live dropdowns; the tool generates the transformation XML, no manual coding needed per project
  • Jakarta Mail integration — HTML progress reports every 10 records and a final completion email with per-document pass/fail details
  • Both tools built as thin desktop applications with minimal dependencies, designed for direct deployment in customer environments

Test Automation Executor — Auto Orbit Built at Tiera
Desktop Execution Agent · Web Recorder · Multi-browser · Server-driven

A JavaFX + Spring Boot desktop client designed to be a local test execution and recording agent. It connects to a central Auto Orbit backend server to receive browser test suites and run them locally via Selenium — test definitions live entirely on the server. The same tool also records new test cases by watching and capturing real user interactions in a live browser.

  • Server-driven execution model — fetches step-by-step test instructions (action, locator type, locator value, input) via JWT REST API and executes them recursively page-by-page until the server reports test completion
  • Supports Chrome, Firefox, and Edge via WebDriverManager; handles all Selenium locator strategies (id, name, xpath, css, className, linkText, partialLinkText) and actions including click, input, validate, select, hover, clear
  • Playwright-based recorder — the tool launches Chromium, injects a custom JavaScript recorder, streams captured interaction events back to Java via Playwright's exposeBinding bridge, and saves them as test cases to the server
  • Screen video recording (Monte Media) and screenshot capture on pass/fail; results reported back to the backend server for centralised reporting
  • Role-based UI — admin users access the recorder panel; regular users see only the execution panel

XML-Driven RPA Tool — Auto Orbit RPA Built at Tiera
Process Automation Engine · Web · AS400 Terminal · Database · File Generation

A process automation engine designed to let teams define and run multi-system automation workflows entirely through XML — no coding required per workflow. A single XML process file can orchestrate web automation, IBM AS400 terminal interaction, database queries, and document generation in one coordinated sequence.

  • Web step executor (Selenium) — handles click, input, navigate, validate, screenshot steps across any supported locator strategy
  • IBM AS400/i terminal step executor via TN5250J — the tool launches a live TN5250 emulator, handles login, navigates menus, and fills fields programmatically via screen.sendKeys()
  • Database step executor — SQL read/write/insert operations with results piped into the variable context for use in downstream steps
  • File step executors — Excel and PowerPoint (Apache POI), CSV read with quoted-field parsing, PDF generation from HTML templates (OpenHTMLtoPDF + PDFBox), math expression evaluation (exp4j)
  • Full XML control flow: for/foreach loops over data tables, if/elseif/else conditionals on the variable context, per-step continueOnError flags
  • Windows focus manager using JNA Win32 API (EnumWindows, SetForegroundWindow) — switches OS focus between browser and AS400 terminal during multi-application workflow steps
